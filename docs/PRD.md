# 树图探索器 — 产品需求文档

## 1. 产品概述

### 1.1 背景

业务系统中存在大规模的树形/有向无环图（DAG）数据结构，单个节点可能拥有多个父节点（如"哈士奇"同时属于"犬科"和"工作动物"）。当节点总量超过 1000 时，用户需要高效地浏览、搜索和选取目标节点。

### 1.2 目标用户

- 内部运营人员、数据管理员
- 需要在层级分类体系中定位和选取节点的业务用户

### 1.3 核心目标

- 支持 1000+ 节点量级的高效浏览与选取
- 解决 DAG 多父节点场景下的路径展示问题
- 兼顾前后端性能

---

## 2. 功能需求

### 2.1 搜索

#### 2.1.1 全局搜索

- 顶部搜索框，支持模糊匹配
- 快捷键 `Ctrl+K` / `Cmd+K` 聚焦搜索框
- 输入即搜，300ms debounce
- 搜索结果以下拉列表展示，最多展示 30 条
- 搜索走后端接口，前端只负责展示

#### 2.1.2 搜索结果展示

每条搜索结果展示：
- **节点名称**（匹配关键词高亮）
- **父路径列表**（面包屑格式，如 `动物界 > 脊索动物门 > 哺乳纲 > 食肉目 > 犬科 > 哈士奇`）

#### 2.1.3 多父路径收折

当节点拥有 3 条及以上父路径时，触发路径收折：

| 条件 | 行为 |
|------|------|
| 路径数 ≤ 2 | 全部展示，用 ` | ` 分隔 |
| 路径数 ≥ 3 | 默认展示最短的 2 条路径，下方显示 `+N 条其他路径 ▸` |

收折交互规则：
- 折叠按钮使用品牌主色
- 点击展开后，剩余路径用虚线分隔，带左侧缩进和降低透明度
- 展开后按钮文案变为 `▾ 收起`，再次点击折叠
- 路径按深度升序排列（最短路径优先展示）

#### 2.1.4 选中行为

- 点击搜索结果：选中该节点，关闭搜索下拉，左侧树自动展开到该节点并高亮
- 搜索框 `Escape` 关闭下拉，保留输入内容
- 点击搜索框外部关闭下拉

---

### 2.2 树面板（左侧）

#### 2.2.1 层级浏览

- 树节点可折叠/展开，初始只展开前 2 层
- 叶节点无展开箭头，有展开箭头的节点表示存在子节点
- 当前选中节点以品牌浅色背景高亮

#### 2.2.2 聚合节点

当某节点的直接子节点数超过 50 时，自动折叠为聚合节点：

- 展示为 `▸ N 个子节点`，附带计数徽标
- 点击聚合节点后展开其直接子节点列表
- 聚合节点的子节点如果也超过 50 个，继续聚合

#### 2.2.3 懒加载（渐进式展开）

- 初始只渲染根节点和前 2 层
- 用户展开节点时，按层请求该节点的直接子节点
- 已展开过的子树缓存到本地，回折时不重复请求
- 展开时展示 loading 状态

---

### 2.3 详情面板（右侧）

选中节点后展示以下信息：

| 区域 | 内容 |
|------|------|
| 节点名称 | 大号标题 |
| 所属路径 | 所有父路径面包屑（同搜索结果，3+ 条路径时应用收折） |
| 子节点数 | 直接子节点数量 |
| 父节点数 | 直接父节点数量 |
| 描述 | 节点描述文本 |

未选中节点时展示空状态提示。

---

## 3. 交互规范

### 3.1 键盘快捷键

| 快捷键 | 行为 |
|--------|------|
| `Ctrl+K` / `Cmd+K` | 聚焦搜索框 |
| `Escape` | 关闭搜索下拉 |
| `↑` / `↓` | 搜索结果中移动焦点 |
| `Enter` | 确认选中当前焦点项 |

### 3.2 状态反馈

- 树节点：hover 浅灰背景，选中品牌浅色背景
- 搜索结果：hover 浅灰背景，焦点品牌浅色背景
- 展开/收起箭头：90° 旋转动画，160ms
- 路径收折：展开/收起无动画，即时切换

---

## 4. 性能需求

### 4.1 前端

| 策略 | 说明 |
|------|------|
| 懒加载 | 按层请求子节点，不一次性拉取全量数据 |
| 本地缓存 | 已展开的子树缓存到内存，避免重复请求 |
| DOM 控制 | 懒加载模式下 DOM 节点数可控；若全量渲染则使用虚拟滚动 |
| 防抖 | 搜索输入 300ms debounce |
| 请求取消 | 快速连续输入时取消上一次未完成的请求（AbortController） |

### 4.2 后端

| 策略 | 说明 |
|------|------|
| 分层查询 | `GET /nodes/:id/children?size=50`，不返回整棵树 |
| 搜索索引 | 全文索引支撑模糊搜索 |
| 多父路径 | 递归 CTE 或图数据库查询祖先路径 |
| 热点缓存 | 高频访问的子树（如前 3 层）Redis 缓存 |

## 5. 接口设计

### 5.1 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/nodes/root` | GET | 获取根节点列表 |
| `/nodes/:id/children?size=50` | GET | 懒加载子节点 |
| `/nodes/search?q=keyword&limit=30` | GET | 全局搜索 |
| `/nodes/:id/paths` | GET | 获取所有父路径（独立调用场景） |

### 5.2 接口详情

#### 获取根节点

```
GET /nodes/root
Response: [{ id, name, childCount }]
```

#### 懒加载子节点

```
GET /nodes/:id/children?size=50
Response: [{ id, name, childCount, parentIds[] }]
```

- `childCount` 用于前端判断是否需要展示聚合节点
- `parentIds` 用于多父节点关系的路径计算

#### 全局搜索

```
GET /nodes/search?q=keyword&limit=30
Response: [{
  id,
  name,
  paths: ["动物界 > 脊索动物门 > ... > 哈士奇", "功能分类 > 工作动物 > 哈士奇"]
}]
```

- `paths` 直接在搜索接口中返回，减少前端额外请求
- 路径按深度升序排列

#### 获取所有父路径

```
GET /nodes/:id/paths
Response: { paths: [["根 > A > B", "根 > C > B"]] }
```

- 适用于选中节点后详情面板展示、或搜索接口未返回路径时的补充查询

---

## 6. 数据模型

### 6.1 节点结构

```
Node {
  id: number
  name: string
  parentIds: number[]    // 多父节点，DAG 关系
  description: string
}
```

### 6.2 多父节点说明

一个节点可以同时属于多个父节点，形成 DAG（有向无环图）而非纯树。典型场景：

```
犬科 ──→ 哈士奇 ←── 工作动物
                  ←── 展览动物
                  ←── 伴侣动物
                  ←── 农业动物
```

"哈士奇"拥有 4 个父节点，对应 4 条从根到叶的祖先路径。

## 7. 布局规格

### 7.1 整体布局

- 桌面端左右双栏：左侧树面板 360px 固定宽度，右侧详情面板弹性宽度
- 顶部搜索栏 48px 高度
- 整体占满视口高度（100vh）

### 7.2 树面板

- 每行节点高度 28px
- 展开箭头 20x20px
- 层级缩进每级 20px
- 聚合节点计数徽标 18px 高度

### 7.3 搜索下拉

- 最大高度 380px，超出滚动
- 每条结果：节点名 + 路径列表
- 路径文字 11px，节点名 13px

## 8. 非功能需求

| 项目 | 要求 |
|------|------|
| 首屏加载 | 仅加载根节点和前 2 层，首屏请求数 ≤ 3 |
| 搜索响应 | 300ms 内返回结果 |
| 节点展开 | 单层子节点加载 ≤ 200ms |
| 兼容性 | 主流桌面浏览器（Chrome、Firefox、Safari、Edge） |
| 可访问性 | 支持键盘导航、ARIA 属性、焦点可见状态 |
